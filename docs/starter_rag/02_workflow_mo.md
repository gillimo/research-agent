Workflow MO
===========

Session start
1) Check `git status -sb`.
2) Review `docs/tickets.md` for priorities.
3) Clock in via `/clock in`.

During work
- Make changes in small batches.
- Update docs when behavior changes.
- Log bugs in `docs/bug_log.md`.
- Run tests relevant to the change.

Session end
1) Provide `/signoff` summary.
2) Clock out via `/clock out`.
3) Ensure tickets are updated.
